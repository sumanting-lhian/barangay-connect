Buttons

Every interactive clickable element in BarangayConnect. Always start with the base .btn class, then layer a variant, size, and modifier. Requires main.css + buttons.css.

Tab / Category Pills

.btn--category + .btn-tab-bar

Primary navigation tabs for dark backgrounds — dashboard hero strips, community section tabs, service quick-action bars. Wrap in .btn-tab-bar for correct spacing. Active state uses .is-active (or .is-selected). The orange fill on active is intentional: it creates visible contrast against green hero sections. Use .btn--filter instead when the background is white or off-white.

Services strip — Report a Concern active 
<div class="btn-tab-bar">
  <button class="btn btn--category is-active">
    <i data-lucide="flag"></i> Report a Concern
  </button>
  <button class="btn btn--category">
    <i data-lucide="file-text"></i> Document Requests
  </button>
</div>

/* Toggle active tab with JS */
document.querySelectorAll('.btn--category').forEach(btn => {
  btn.addEventListener('click', function() {
    this.closest('.btn-tab-bar')
      .querySelectorAll('.btn--category')
      .forEach(b => b.classList.remove('is-active'));
    this.classList.add('is-active');
  });
});

Widget Buttons

.btn--widget

Glass-style icon + label tiles for the time/alert widget overlay on the dashboard hero. Always used on dark or photo backgrounds — the backdrop-filter: blur gives the frosted effect. Stack icon above label, keep label text short.

Dashboard hero overlay 
<button class="btn btn--widget">
  <i data-lucide="file-text"></i>
  <span>Docs</span>
</button>

Solid Buttons

.btn--green / --orange / --red

Primary actions on white and light backgrounds. Green = confirm / register / official action. Orange = submit / RSVP / call-to-action. Red = destructive / urgent / emergency. All three have built-in hover, active, and disabled states.

All solid variants — with icons and without 
<button class="btn btn--green">Register</button>
<button class="btn btn--green">
  <i data-lucide="phone"></i> Contact Owner
</button>
<button class="btn btn--orange">RSVP Now</button>
<button class="btn btn--red">Report Now</button>
<button class="btn btn--green" disabled>Disabled</button>

Outline & Ghost

.btn--outline / --ghost / --dashed / --white

Secondary and tertiary actions. Use .btn--outline for Back/Cancel on light backgrounds. .btn--ghost for subtle secondary links. .btn--dashed for invite / add-new actions. .btn--white is a solid white pill, used on dark hero sections only.

On light backgrounds 
<button class="btn btn--outline"><i data-lucide="chevron-left"></i> Back</button>
<button class="btn btn--outline">Cancel</button>
<button class="btn btn--ghost btn--sm">About</button>
<button class="btn btn--dashed">Request New Document</button>

Hero Buttons

.btn--outline-hero / --white

For use on dark green hero sections and banner areas. .btn--outline-hero is a white-bordered ghost — turns solid white on hover. .btn--white is always solid white. Pair with .btn--rounded for the softer pill shape seen on landing sections.

On dark hero backgrounds 
<!-- Ghost white outline, turns solid on hover -->
<button class="btn btn--outline-hero btn--lg btn--rounded">
  Get Started <i data-lucide="arrow-right"></i>
</button>

<!-- Always solid white -->
<button class="btn btn--white btn--lg">Browse Community</button>

<!-- Orange works great on dark sections too -->
<button class="btn btn--orange btn--full">Submit Concern</button>

Location Reported

.btn--located

Replaces the "Contact Owner" button on a pet card once someone has confirmed a sighting. Non-interactive — always add the disabled attribute. The muted green-gray color communicates "resolved, no action needed."

Close / Dismiss

.btn--close

Circular X button for modals, overlays, notification panels, and dismissible chips. Always add aria-label="Close". Use .btn--sm for tighter contexts like inline chips or small confirmation toasts. The red background is intentional — it makes the close action immediately obvious even without reading the icon.

Default and small
Default (40px)
Small (28px) 
<button class="btn btn--close" aria-label="Close">
  <i data-lucide="x"></i>
</button>
<!-- Small: add .btn--sm -->

Loading State

.is-loading + .btn__spinner

Add .is-loading to any .btn to show the spinner. Include <span class="btn__spinner"></span> inside the button. The button becomes non-interactive while loading. Works on all solid and outline variants.

Loading on orange and green 
<button class="btn btn--orange is-loading">
  <span class="btn__spinner"></span>
  Submitting...
</button>

/* JS: toggle loading on form submit */
btn.classList.add('is-loading');
// after async work finishes:
btn.classList.remove('is-loading');

Call Buttons

.btn--call / --call-circle

Two styles for phone contact actions. .btn--call-circle is an icon-only orange circle — used on pet board cards and the local services directory. .btn--call is a pill with a label — used on emergency contact rows where the category label helps identify who you're calling.

Toggle Switch

.toggle-wrap + .toggle-track + .toggle-thumb

Pure CSS toggle — no JS needed for the visual. The checkbox manages checked state; CSS does the rest. Green track = active/on. Orange track (.toggle-track--orange) for "Featured" or alert toggles. Always wrap with .toggle-wrap and include a visible .toggle-label for accessibility.

All toggle variants — off / green / orange / disabled
Off
On — green
On — orange
Disabled 
<label class="toggle-wrap">
  <input type="checkbox" class="toggle-input" />
  <span class="toggle-track">
    <span class="toggle-thumb"></span>
  </span>
  <span class="toggle-label">Active</span>
</label>

<!-- Orange track for "Featured" / alert toggles -->
<span class="toggle-track toggle-track--orange">
  <span class="toggle-thumb"></span>
</span>

Filter Pills

.btn--filter

Content filter pills for light backgrounds — bulletin feed, admin tables, concern lists. Inactive = gray, active = green. Do not use on dark hero sections; use .btn--category there instead.

Light background filter bar 
<button class="btn btn--filter is-active">All</button>
<button class="btn btn--filter">Announcements</button>
<button class="btn btn--filter">Reports</button>

/* Toggle active filter */
document.querySelectorAll('.btn--filter').forEach(btn => {
  btn.addEventListener('click', function() {
    this.closest('div')
      .querySelectorAll('.btn--filter')
      .forEach(b => b.classList.remove('is-active'));
    this.classList.add('is-active');
  });
});

Icon-Only Buttons

.btn--icon + .btn--icon-edit / --icon-delete

Compact action buttons for table rows, card action menus, and admin panels. Always include aria-label — there is no visible text. Use .btn--sm to shrink from 40px to 32px for tighter layouts.

Floating Action Button

.btn--fab

Fixed bottom-right. One per page maximum. Use for the primary creation action on a page — "Create post", "New report", "Add event". The position: fixed is built into the class; it renders statically below in the demo.

FAB — static for demo, fixed in production 
<!-- Place before </body> — position: fixed is built into the class -->
<button class="btn btn--fab" aria-label="Create new post">
  <i data-lucide="plus"></i>
</button>

Selector Cards

.btn--selector

Large clickable choice cards for multi-step forms. Used on concern type selection, document type, and ID type steps. Toggle .is-selected to highlight the choice. Supports multi-select — toggle each independently.

Concern type selector — Safety Hazard selected 
<button class="btn btn--selector">
  <i data-lucide="droplets"></i> Flooding
</button>
<!-- Add .is-selected for the active choice -->
<button class="btn btn--selector is-selected">
  <i data-lucide="shield-alert"></i> Safety Hazard
</button>

Size Variants

.btn--sm / .btn--lg / .btn--full / .btn--rounded

Size modifiers work on every solid and outline variant. .btn--rounded changes the border-radius to 12px — a slightly softer look for landing sections. .btn--full stretches to 100% width of its container.

Size scale 
<button class="btn btn--green btn--sm">Small</button>
<button class="btn btn--green">Default</button>
<button class="btn btn--green btn--lg">Large</button>
<button class="btn btn--orange btn--full">Full Width</button>
<button class="btn btn--green btn--lg btn--rounded">Rounded</button>
